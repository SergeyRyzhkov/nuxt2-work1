1. Общий осмотр на предмент орфографических, синтаксических и стилистических ошибок
   (не Название обучения, а Наименование курса итд)
2. Фильтрация
   - Убедиться, что по каждому полю производится фильтрация списка
   - В поле филтрации отображается введенное значение (бывает вводишь текст, фильтр идет, но не виден)
3. Список
   - Убедиться, что соритровка по кажому полю верная. Числа, даты по убыванию\возрастанию, строки по алфавиту.
     Типовая ошибка, кода числа (даты) выводятся какс троки и сортировка не верная
   - Убедиться, что при пагинации (постраничном переходе) не происходит сброс филтрации и сортировки
   - Таблица должна иметь горизонтальную прокрутку
   - Наименования колонок должны быть "локаничны", не занимая ширину колонки. Количество свободных мест => Свободных мест итд
   - Должна быть возможность указать количество выводимых строк на странице
   - При нажатии кнопки "Удалить" должно быть предупреждение. При этом желательно текст писать утвердительным,
     а не уважительным. не принято выводить "Вы действительно желаете удалить обучение ? :)" => "Удалить курс ?"
4. Карточка (форма) ввода данных
   - Поля ввода должны иметь "пометку" об обязательности заполнения Поля
   - Поля ввода должны иметь маску ввода по типу данных (число, дата) и по виду емайл, ИНН, номер паспорта
   - Поля ввода желательно сразу валидировать после потери фокуса, а не по нажатию Обновить (Сохранить, OK)
   - После валидации поля ввода с незаполненными (или неверным форматом) должны быть обозначены (бордер красный, итд)
   - Сообщения о валидации должны быть на языке проекта (в нашем случае на русском) и иметь для кажого сообщения перенос на следующую строку (не в одну сроку ...каша)
   - После сохранения и удаления данных, убедиться, что в БД и\или соответствующем эндпоинте данные реально изменены и не возникло ошибок (каскадность итд)
